중앙 처리 장치 : 주 메모리 액세스

M1 1 비트

이 값을 사용하면 메모리 값 또는 내부 데이터 버스 값을 사용하여 데이터 레지스터 (MBR)를 업데이트하는 데 사용할 수있는 값을 선택할 수 있습니다.

MH 1 bit

이 신호를 통해 다음과 같이 작업할 하드웨어 성능 카운터(HPC)를 선택할 수 있습니다.

  • 00: 클럭 사이클 수..
  • 01: 누적 CPU 시간..
  • 10: 누적 전력 소비.
  • 11: <미사용>.

하드웨어 성능 카운터는 하드웨어 관련 활동에 대한 통계를 저장하기 위한 특수 목적 레지스터 세트입니다. 낮은 수준의 성능 또는 전력 소비 분석을 수행하는 데 사용됩니다.

R 1 비트
W 1 비트
MRdy 1 비트
BW 2 비트

주 메모리의 제어 신호 : R은 읽기, W는 쓰기.
BW는 읽기/쓰기 할 바이트 수를 제어합니다. 1 바이트는 0, 2 바이트 (하프 워드)는 1, 4 바이트 (워드)는 3입니다.
기본적으로 메모리 읽기 또는 쓰기는 1 클럭 사이클로 수행됩니다. 메모리가 수 클록 사이클 동안 구성되면 메모리가 R/W 요청을 끝내면 MRdy 신호가 설정됩니다.

TEST_C, ...는 ALU에 의해 생성되므로 직접 사용하지 마십시오.

사용자가 하드웨어 요소에 의해 생성 된 플래그의 값을 즉시 변경할 수있게하십시오.
현재 사용 가능한 플래그는 다음과 같습니다.

  • C : 캐리
    • a 추가 : (& gt; & gt; & gt; 31) && (b> & gt; & gt; 31)
  • V : oVerflow
    • a 추가 : (a + b <0) && (a> = 0) && (b> = 0) || (a + b & gt; = 0) && (a & lt; 0) && (b & lt; 0)
  • N : 제외 어
    • a 추가 : (a + b <0)
  • Z : 제로
    • a 추가 : (a + b == 0)

  • I : 중단 활성화/중단
  • U : 사용자 수준/커널 수준
