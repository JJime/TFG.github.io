A unidade central de processamento: acesso à memória principal

M1 1 bit

Eles permitem que você selecione qual valor pode ser usado para atualizar o registrador de dados (MBR): com o valor da memória ou o valor do barramento de dados interno.

MH 1 bit

Este sinal permite selecionar o Hardware Performance Counter (HPC) para trabalhar com:

  • 00: Número de ciclos de clock.
  • 01: Tempo de CPU acumulado.
  • 10: Consumo de Energia Acumulado.
  • 11: <Não usado>.

O Contador de desempenho de hardware é um conjunto de registros de finalidade especial para armazenar estatísticas sobre atividades relacionadas a hardware. Ele é usado para realizar análises de desempenho ou consumo de energia de baixo nível.

R 1 bit
W 1 bit
MRdy 1 bit
BW 2 bits

Sinais de controle da memória principal: R para leitura, W para gravação.
BW controla o número de bytes a serem lidos/gravados: 0 para um byte, 1 para dois bytes (meia palavra) e 3 para quatro bytes (uma palavra).
Por padrão, a memória lê ou grava em um ciclo de clock. Se a memória estiver configurada para vários ciclos de clock, o sinal MRdy será definido quando a memória terminar a solicitação de R/W.

A unidade central de processamento: registra

Ele irá armazenar no registro associado, o valor da entrada do registro no final do ciclo do relógio.

A unidade central de processamento: Acesso ao barramento de endereços

A unidade central de processamento: Acesso ao barramento de dados interno

T1, ..., T12 1 bit

Copie o valor na entrada do elemento tri-state associado no barramento de dados interno, durante o ciclo do relógio.

A unidade central de processamento: os sinalizadores ALU, mais os sinalizadores I e U

NOTA: este mecanismo é apenas para fins de teste. Você não deve usar esses signals na versão final do seu microcódigo.
Por favor, evite usar TEST_C, ... diretamente porque seu valor é gerado pela ALU.

TEST_C, ..., TEST_U 1 bit

Permita que o usuário altere os valores dos sinalizadores enquanto são gerados pelos elementos de hardware.
Os sinalizadores disponíveis atuais são:

  • C : carregar
    • Adicione um b : (a & gt; & gt; & gt; 31) && (b & gt; & gt; & gt; 31)
  • V : oVerflow
    • Adicione um b : (a + b & lt; 0) && (a & gt;=0) && (b & gt;=0) || (a + b & gt;=0) && (a & lt; 0) && (b & lt; 0)
  • N : negativo
    • Adicione um b : (a + b & lt; 0)
  • Z : zero
    • Adicione um b : (a + b == 0)

  • I : Interrupção ativada/Interrupção desativada
  • U : nível do usuário/nível do kernel

ExCode 4 bits

Deixe o microprogramador especificar um valor inmediate que deve ser copiado no barramento de dados interno com o sinal T11.

Os elementos de entrada/saída

IOR 1 bit
IOW 1 bit
IORdy 1 bit

Sinais de controle para os dispositivos de E/S: IOR para leitura, IOW para gravação e, se os dispositivos forem assíncronos, o sinal IORdy é definido quando o dispositivo finaliza a solicitação.
